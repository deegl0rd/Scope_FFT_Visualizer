# Build Instructions
This project was developed for the STM32F429I-DISC1 development board using STM32CubeIDE and TouchGFX.

# Requirements
## Hardware
- STM32F429I-DISC1 development board
- USB cable
- Optional:
  - Signal generator
  - Breadboard analog frontend
  
# Required Software

## STM32CubeIDE
Recommended version:

```text
STM32CubeIDE 2.1.1 or newer
```

Download:

- https://www.st.com/en/development-tools/stm32cubeide.html

## TouchGFX
Recommended version:

```text
TouchGFX 4.26.1
```

Download:

- https://support.touchgfx.com/

# Required Middleware
The project uses the CMSIS DSP library through:

```text
X-CUBE-ALGOBUILD
```

Install it through STM32CubeMX if missing.

# Importing the Project
## 1. Clone Repository

```bash
git clone <repository-url>
```

## 2. Open STM32CubeIDE

Select:

```text
File → Import → Existing Projects into Workspace
```

Then select the cloned repository folder.

## 3. Generate TouchGFX Code
Open:

```text
TouchGFXDesigner
```

Then click:

```text
Generate Code
```

This regenerates TouchGFX generated sources.

## 4. Build Project
Inside STM32CubeIDE:

```text
Project → Build Project
```

or press:

```text
CTRL + B
```

# Flashing the Board
Connect the STM32F429I-DISC1 board via USB.

Then:

```text
Run → Debug
```

or

```text
Run → Run
```
STM32CubeIDE automatically flashes the firmware using ST-LINK.

# Important Notes
## TouchGFX Generated Files
Some files are automatically generated by TouchGFX and CubeMX.

If build issues occur:
1. Open `.ioc`
2. Regenerate code
3. Rebuild project

## Portable Paths
The project uses workspace-relative paths:

```text
${workspace_loc:/${ProjName}/...}
```
to ensure portability across systems.

# Troubleshooting
## Missing DSP Library
If `arm_math.h` is missing:

- Install `X-CUBE-ALGOBUILD`
- Regenerate CubeMX project files
## TouchGFX Build Errors

Open TouchGFXDesigner and regenerate code.
